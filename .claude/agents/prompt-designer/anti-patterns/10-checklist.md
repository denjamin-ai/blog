# Countermeasures Application Checklist

When creating or updating rules, verify:

- [ ] Critical rules repeated at beginning AND end?
- [ ] Tasks broken into stages (not "everything at once")?
- [ ] Rules are operational (not abstract)?
- [ ] Explicit priority hierarchy exists?
- [ ] Structure is clear (RFC-style)?
- [ ] Output format is strict (with examples)?
- [ ] Summarize-first for long contexts?
- [ ] External data explicitly marked as UNTRUSTED?
- [ ] "I don't know" policy exists?
- [ ] No "magical thinking" (everything explicit)?
- [ ] No "Caesar" agents (one agent doing multiple unrelated functions)?
