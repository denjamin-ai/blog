# 4. Chain-of-Thought / Think-First Pattern

**What**: Ask the model to think first, then respond.

**When**: Complex tasks: architecture, integrations, refactoring.

**Application**:
- Instruction to think step-by-step before generating code
- Break down into subtasks
- Internal reasoning before response
