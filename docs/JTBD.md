# JTBD — Jobs to Be Done

Полное покрытие пользовательских историй по каждому типу пользователя. Формат: **Когда** хочу → **Инструмент** (что нажимаю, в каком порядке) → **Успех / Ошибка**.

---

## ГОСТЬ (не аутентифицирован)

### US-G1: Хочу узнать, о чём этот блог
- **Инструмент:** Переходит на `/`
- **Успех:** Видит аватар, имя, bio автора блога; 3 последние опубликованные статьи; кнопку «Все статьи»
- **Ошибка:** Профиль не заполнен → пустые поля, страница рендерится; сервер упал → 500

### US-G2: Хочу найти и прочитать статью
- **Инструмент:** `/blog` → просматривает карточки → кликает на карточку → `/blog/[slug]`
- **Успех:** Карточки с заголовком, описанием, тегами, датой, бейджем сложности; полный MDX-контент на странице статьи
- **Ошибка:** Статья не найдена или автор заблокирован → 404

### US-G3: Хочу посмотреть историю изменений статьи
- **Инструмент:** На странице `/blog/[slug]` разворачивает компонент `ArticleChangelog`
- **Успех:** Список записей с датой, секцией, описанием изменений
- **Ошибка:** Записей нет → компонент не отображается

### US-G4: Хочу скопировать код из статьи
- **Инструмент:** Нажимает иконку копирования на блоке кода (`CopyButton`)
- **Успех:** Код скопирован в буфер, иконка меняется на «✓»
- **Ошибка:** Clipboard API недоступен → кнопка не реагирует

### US-G5: Хочу прочитать публичные комментарии к статье
- **Инструмент:** Прокручивает страницу статьи вниз до секции комментариев
- **Успех:** Дерево комментариев (до 2 уровней), имя автора, дата, текст; пагинация «Загрузить ещё»; удалённые показываются как «[удалено]»
- **Ошибка:** Ошибка загрузки → пустой блок или сообщение об ошибке

### US-G6: Хочу оставить комментарий
- **Инструмент:** Видит ссылку «Войдите, чтобы оставить комментарий» → кликает → `/login`
- **Успех:** Открывается форма входа для читателей/авторов/ревьюеров
- **Ошибка:** —

### US-G7: Хочу переключить тему оформления
- **Инструмент:** Кликает `ThemeToggle` в шапке
- **Успех:** Тема переключается, предпочтение сохраняется в localStorage
- **Ошибка:** —

### US-G8: Хочу посмотреть публичный профиль автора
- **Инструмент:** Кликает на имя/аватар автора → `/authors/[slug]`
- **Успех:** Страница с отображаемым именем, bio, аватаром, ссылками (GitHub, Telegram, сайт); список опубликованных статей автора
- **Ошибка:** Автор не найден или `slug` не задан → 404; автор заблокирован → его статьи не отображаются

### US-G12: Хочу понять, что можно делать на сайте
- **Инструмент:** Нажимает иконку «Книга» (руководство) в навигации
- **Успех:** Открывается модальное окно с контентом для гостя: чтение статей и блога, TOC, difficulty-бейдж, публичные комментарии (чтение), RSS-лента, профили авторов, как войти или зарегистрироваться; закрывается по Escape / клику на backdrop
- **Ошибка:** —

### US-G9: Хочу подписаться на RSS-ленту
- **Инструмент:** Открывает `/feed.xml` в RSS-ридере
- **Успех:** Валидный RSS 2.0 со всеми опубликованными статьями, именами авторов (`dc:creator`), тегами как категориями; кеш 1 час
- **Ошибка:** —

### US-G10: Хочу посмотреть встроенное видео в статье
- **Инструмент:** Читает статью на `/blog/[slug]`
- **Успех:** Видео воспроизводится через HTML5 `<video>` с controls; без автозапуска; адаптивная ширина
- **Ошибка:** Браузер не поддерживает формат → стандартный fallback-текст браузера

### US-G11: Хочу увидеть диаграмму в статье
- **Инструмент:** Читает статью на `/blog/[slug]`, содержащую `<Mermaid>` или `<Diagram>`
- **Успех:** Mermaid-диаграмма рендерится на клиенте (lazy, при попадании в viewport); Kroki-диаграмма отображается как SVG (серверный рендер); цвета соответствуют выбранной теме; при ошибке синтаксиса Mermaid — текстовое сообщение об ошибке; при недоступности kroki.io — исходный код в `<pre>`
- **Ошибка:** Kroki.io недоступен → fallback `<pre><code>` с исходным кодом диаграммы

## ЧИТАТЕЛЬ (role: reader)

### US-R1: Хочу войти в систему
- **Инструмент:** `/login` → вводит никнейм + пароль → «Войти»
- **Успех:** Редирект на `/`, сессия с `role=reader`
- **Ошибка:** Неверные данные → сообщение об ошибке на форме

### US-R2: Хочу оставить комментарий к статье
- **Инструмент:** На `/blog/[slug]` → вводит текст в форму → «Отправить»
- **Успех:** Комментарий появляется в списке
- **Ошибка:** `commentingBlocked=1` → 403; статья снята с публикации → 403; пустой текст → клиентская валидация

### US-R3: Хочу ответить на чужой комментарий
- **Инструмент:** «Ответить» под комментарием → вложенная форма → «Отправить»
- **Успех:** Ответ появляется с отступом под родительским
- **Ошибка:** Уже 2 уровня вложенности → кнопка «Ответить» отсутствует; ошибка сети → сообщение

### US-R4: Хочу отредактировать свой комментарий
- **Инструмент:** На свежем комментарии (< 15 мин) → «Редактировать» → редактирует текст → сохраняет
- **Успех:** Текст обновился, появляется пометка «(ред.)»
- **Ошибка:** Окно 15 мин истекло → 403; ошибка сети → сообщение

### US-R5: Хочу удалить свой комментарий
- **Инструмент:** «Удалить» на своём комментарии → подтверждение
- **Успех:** Комментарий заменяется на «[удалено]», ветка дочерних остаётся
- **Ошибка:** —

### US-R6: Хочу видеть предупреждение о старой версии статьи
- **Инструмент:** Открывает статью, если текущая версия отличается от версии, к которой привязан комментарий
- **Успех:** Рядом с комментарием виден компонент `VersionWarning` с датой старой версии
- **Ошибка:** —

### US-R7: Хочу выйти из системы
- **Инструмент:** Кнопка выхода в навигации → вызов `DELETE /api/auth`
- **Успех:** Сессия очищена, при переходе на `/blog` — гость
- **Ошибка:** —

### US-R8: Хочу проголосовать за статью
- **Инструмент:** На странице статьи нажимает кнопку «+1» или «−1» (`ArticleVoting`)
- **Успех:** Голос учитывается; повторное нажатие той же кнопки отменяет голос; нажатие противоположной — переключает
- **Ошибка:** `commentingBlocked=1` → 403; голос за свою статью → 403; rate-limit (1 req/s) → 429

### US-R9: Хочу сохранить статью в закладки
- **Инструмент:** Нажимает `BookmarkButton` на карточке или странице статьи
- **Успех:** Статья добавлена в закладки; кнопка меняет состояние; публичный счётчик закладок виден всем посетителям
- **Ошибка:** Не авторизован → 401; ошибка сети → сообщение

### US-R10: Хочу просмотреть свои закладки
- **Инструмент:** Переходит на `/bookmarks`
- **Успех:** Список сохранённых статей с заголовком, аннотацией, тегами, датой добавления; кнопка «Удалить из закладок»; отсортированы от новых к старым
- **Ошибка:** Закладок нет → пустое состояние

### US-R11: Хочу подписаться на автора
- **Инструмент:** На странице `/authors/[slug]` → кнопка `SubscribeButton`
- **Успех:** Подписка оформлена; повторное нажатие — отписка; `GET /api/subscriptions` возвращает список подписок
- **Ошибка:** Не авторизован → 401; ошибка сети → сообщение

### US-R12: Хочу проголосовать за комментарий
- **Инструмент:** Нажимает «+1» / «−1» рядом с комментарием
- **Успех:** Голос учитывается; toggle/switch аналогично статьям; нельзя голосовать за свой комментарий
- **Ошибка:** `commentingBlocked=1` → 403; rate-limit → 429

### US-R14: Хочу открыть руководство по возможностям читателя
- **Инструмент:** Нажимает иконку «Книга» в навигации
- **Успех:** Открывается модальное окно с контентом для роли «Читатель»: оставление комментариев (2 уровня вложенности, 15-мин редактирование), голосование за статьи и комментарии, закладки, подписка на авторов, лента подписок, уведомления; закрывается по Escape / backdrop
- **Ошибка:** —

### US-R13: Хочу получать уведомления о событиях
- **Инструмент:** Значок `NotificationBadge` → `/notifications`
- **Успех:** Список уведомлений: `new_article_by_subscribed_author` (новая статья от автора, на которого подписан), `article_updated_for_subscribers` (обновление статьи), `public_comment_reply` (ответ на комментарий); непрочитанные выделены; «Отметить все прочитанными»
- **Ошибка:** —

## АВТОР (role: author)

### US-A1: Хочу войти в систему
- **Инструмент:** `/login` → никнейм + пароль → «Войти»
- **Успех:** Редирект на `/author`
- **Ошибка:** Неверные данные → ошибка на форме

### US-A2: Хочу видеть обзор своей деятельности
- **Инструмент:** Переходит на `/author`
- **Успех:** 3 карточки: Всего / Опубликовано / Черновики; кнопки «Написать статью», «Все мои статьи»
- **Ошибка:** —

### US-A3: Хочу создать новую статью (черновик)
- **Инструмент:** «Новая статья» → `/author/articles/new` → заполняет Заголовок (слаг генерируется), Описание, Теги, MDX-контент → «Сохранить черновик»
- **Успех:** Статья создана как `draft`, редирект на `/author/articles/[id]`
- **Ошибка:** Слаг занят → ошибка; пустой заголовок → валидация; ошибка сети → сообщение

### US-A4: Хочу опубликовать статью
- **Инструмент:** На `/author/articles/[id]` → «Опубликовать»
- **Успех:** Статья становится `published`, `publishedAt` проставляется, кнопка меняется на «Снять с публикации»
- **Ошибка:** `isBlocked=1` → 403; ошибка сети

### US-A5: Хочу снять статью с публикации
- **Инструмент:** На `/author/articles/[id]` → «Снять с публикации»
- **Успех:** Статья переходит в `draft`, исчезает с публичного блога
- **Ошибка:** —

### US-A6: Хочу отредактировать статью
- **Инструмент:** `/author/articles` → «Редактировать» → меняет поля → вводит заметку об изменениях → «Сохранить»
- **Успех:** Версия старого содержимого сохраняется в `articleVersions`, статья обновляется
- **Ошибка:** Слаг занят другой статьёй → ошибка; чужая статья → 403

### US-A7: Хочу отправить статью на ревью
- **Инструмент:** На редактировании статьи → «На ревью» → `ReviewerPickerModal` → вводит 2+ символа → выбирает 1–3 ревьюеров через чекбоксы → подтверждает отправку
- **Успех:** Создаётся `reviewSession` (open) + отдельный `reviewAssignment` (pending) для каждого выбранного ревьюера; все ревьюеры получают уведомление `assignment_created`; на странице статьи появляется блок сессии с карточками всех ревьюеров и их статусами
- **Ошибка:** Активная сессия для этой статьи уже существует → 409; количество ревьюеров ниже минимума для сложности (`medium` требует ≥1, `hard` требует ≥2) → валидация в UI (кнопка заблокирована с пояснением); ревьюер не найден → пустой список

### US-A8: Хочу уведомить ревьюера об изменениях в статье
- **Инструмент:** На `/author/articles/[id]` (активное назначение) → «Уведомить ревьюера»
- **Успех:** Ревьюер получает уведомление `article_updated`
- **Ошибка:** Нет активного назначения → кнопка отсутствует

### US-A9: Хочу посмотреть историю версий статьи
- **Инструмент:** «История» рядом со статьёй → `/author/articles/[id]/history`
- **Успех:** Хронологический список версий: дата, заметка, превью MDX
- **Ошибка:** Нет версий → пустой список

### US-A10: Хочу посмотреть статус ревью
- **Инструмент:** «Ревью» рядом со статьёй → `/author/articles/[id]/review`
- **Успех:** Список всех ревью-назначений: ревьюер, статус (цветной бейдж), дата, заметка ревьюера
- **Ошибка:** Нет назначений → пустой список

### US-A11: Хочу удалить статью
- **Инструмент:** На `/author/articles/[id]` → «Удалить» → подтверждение
- **Успех:** Статья удалена каскадно (версии, комментарии, назначения), редирект на `/author/articles`
- **Ошибка:** Чужая статья → 403

### US-A12: Хочу увидеть уведомления
- **Инструмент:** Значок `NotificationBadge` в навигации → `/author/notifications`
- **Успех:** Непрочитанные (акцентный фон, синяя точка) + прочитанные; типы: `assignment_accepted`, `assignment_declined`, `review_completed`, `review_comment_reply`, `review_comment_reopened`, `article_updated`, `article_hidden`; кнопка «Отметить все прочитанными»; пагинация
- **Ошибка:** —

### US-A13: Хочу отметить уведомление как прочитанное
- **Инструмент:** Кликает на уведомление
- **Успех:** Уведомление помечается как прочитанное, переход к связанному ресурсу
- **Ошибка:** —

### US-A14: Хочу выйти из системы
- **Инструмент:** «Выйти» в навигации `/author/`
- **Успех:** Сессия очищена, редирект на `/login`
- **Ошибка:** —

### US-A15: Хочу отредактировать публичный профиль
- **Инструмент:** «Профиль» в навигации → `/author/profile` → редактирует отображаемое имя, bio, slug, ссылки (GitHub, Telegram, сайт) → загружает аватар → «Сохранить»
- **Успех:** Профиль обновлён; кнопка «Смотреть публичный профиль» ведёт на `/authors/[slug]`
- **Ошибка:** Slug занят другим автором → ошибка; неверный формат slug (не a-z0-9-) → валидация; аватар > 2 МБ или неверный формат → ошибка

### US-A16: Хочу загрузить обложку статьи
- **Инструмент:** В форме редактирования статьи → поле «Обложка» → выбирает файл → `POST /api/upload`
- **Успех:** URL `/uploads/[ulid].[ext]` сохраняется в `coverImageUrl`; превью отображается в форме
- **Ошибка:** Файл > 2 МБ → ошибка; неподдерживаемый формат (не JPG/PNG/WebP) → ошибка

### US-A17: Хочу запланировать публикацию статьи
- **Инструмент:** На `/author/articles/[id]` → «Запланировать» → вводит дату и время → подтверждает
- **Успех:** Статья переходит в `scheduled`, `scheduledAt` сохраняется; cron автоматически публикует в указанное время
- **Ошибка:** Дата в прошлом → валидация; `isBlocked=1` → 403

### US-A18: Хочу предпросмотреть статью
- **Инструмент:** На `/author/articles/[id]` → «Предпросмотр» → `/author/articles/[id]/preview`
- **Успех:** Рендер MDX-контента как на публичной странице, без публикации
- **Ошибка:** Чужая статья → 403

### US-A19: Хочу просмотреть diff изменений статьи
- **Инструмент:** На `/author/articles/[id]/review` → «Diff» рядом с назначением
- **Успех:** `GET /api/author/assignments/[id]/diff` возвращает `{ old, new, hasChanges }`; отображается визуальное сравнение версии на момент назначения с текущей
- **Ошибка:** Нет изменений → `hasChanges: false`; назначение не найдено → 404

### US-A20: Хочу пометить комментарий ревью как решённый
- **Инструмент:** На `/author/articles/[id]/review` → «Решить» рядом с комментарием ревьюера
- **Успех:** `PUT /api/review-comments/[id]/resolve` `{ resolved: true }`; комментарий помечается как решённый (`resolvedAt`)
- **Ошибка:** Назначение в статусе `completed`/`declined` → действие заблокировано

### US-A21: Хочу загрузить видео в тело статьи
- **Инструмент:** На `/author/articles/[id]` → кнопка «+ Медиа» или drag-and-drop файла на textarea
- **Успех:** `POST /api/upload` принимает MP4/WebM ≤ 10 МБ и ≤ 45 сек; показывает превью с длительностью; кнопка «Вставить» вставляет `<ArticleVideo src="..." />` в позицию курсора
- **Ошибка:** Файл > 10 МБ или > 45 сек → сообщение об ошибке; неподдерживаемый тип → ошибка

### US-A28: Хочу вставить математическую формулу в статью
- **Инструмент:** На `/author/articles/[id]` → кнопка «∑ Формула» → вводит LaTeX в панели → выбирает inline/block режим → «Вставить формулу»
- **Успех:** Формула вставляется в позицию курсора как `$...$` (inline) или `$$\n...\n$$` (block); KaTeX-превью отображается прямо в панели; быстрые кнопки (∑, ∫, √, α-λ, матрица) добавляют символы в поле ввода
- **Ошибка:** Пустое поле → кнопка «Вставить» заблокирована

### US-A29: Хочу видеть предпросмотр статьи в реальном времени
- **Инструмент:** На `/author/articles/[id]` → toolbar `EditorWithPreview` → кнопка расположения preview (справа/слева/снизу)
- **Успех:** Через 500 мс после изменения текста в textarea preview-панель обновляется: MDX скомпилирован через `POST /api/preview`, формулы рендерятся через KaTeX, код подсвечен через Shiki; skeleton-анимация на время загрузки; ошибки компиляции показываются inline без крэша
- **Ошибка:** Сетевая ошибка → сообщение в панели preview

### US-A30: Хочу переключить расположение preview-панели
- **Инструмент:** Toolbar над редактором → 4 кнопки: без preview / справа / слева / снизу
- **Успех:** Layout применяется немедленно; выбор сохраняется в `localStorage('editor_preview_layout')`; drag handle позволяет изменить соотношение панелей; на мобильных (< 768px) кнопки «справа»/«слева» скрыты
- **Ошибка:** —

### US-A32: Хочу видеть статус сессии ревью с несколькими ревьюерами
- **Инструмент:** На `/author/articles/[id]` → блок «Сессия ревью»

### US-AD33: Хочу открыть руководство по возможностям администратора
- **Инструмент:** Нажимает иконку «Книга» в навигации
- **Успех:** Открывается модальное окно с контентом для роли «Администратор»: управление статьями всех авторов, CRUD пользователей, блокировка авторов/комментаторов, шаблон чеклиста, настройки блога (profile, OG-image, ссылки), отправка на ревью (мультиревьюер), управление сессиями ревью; закрывается по Escape / backdrop
- **Ошибка:** —

### US-AD31: Хочу увидеть диаграмму в опубликованной статье
- **Инструмент:** Просматривает публичную статью на `/blog/[slug]` или предпросмотр
- **Успех:** Mermaid и Kroki-диаграммы корректно рендерятся — аналогично US-G11
- **Ошибка:** Аналогично US-G11

## Критические граничные случаи

| Сценарий | Место проверки |
|----------|---------------|
| `publicComments` RESTRICT при удалении версии | Нужно сначала удалить комментарии — UI должен предупреждать |
| `isBlocked=1` → cascade unpublish всех статей автора | Side effect в `PUT /api/admin/users/[id]` |
| 409 при дублировании назначения (пара articleId+reviewerId) | `POST /api/author/assignments`, `POST /api/admin/assignments` |
| 15-минутное окно редактирования комментария | `PUT /api/articles/[id]/comments/[commentId]` |
| Ревьюер видит версии только до даты назначения | `GET /api/articles/[id]/versions?assignmentId=X` |
| Слаг — уникальность только на уровне приложения, без DB constraint | `PUT /api/articles/[id]`, `POST /api/articles` |
| Автор не может публиковать если `isBlocked=1` | `PUT /api/articles/[id]` с `saveMode=publish` |
| Читатель может редактировать только в первые 15 минут | `PUT /api/articles/[id]/comments/[commentId]` |
| Вердикт обязателен при завершении ревью | `PATCH /api/reviewer/assignments/[id]` со статусом `completed` |
| `commentingBlocked=1` блокирует вотинг (статьи и комментарии) | `POST /api/articles/[id]/votes`, `POST /api/comments/[id]/votes` |
| Slug автора — уникальность на уровне приложения | `PUT /api/author/profile` |
| Разрешение/переоткрытие комментария заблокировано при `completed`/`declined` | `PUT /api/review-comments/[id]/resolve` |
| Дата планирования должна быть в будущем | `PUT /api/articles/[id]` с `saveMode=schedule` |
| Upload: Content-Length проверяется до буферизации (DoS-защита) | `POST /api/upload` |
| Активная сессия уже есть → новую нельзя создать (409) | `POST /api/review-sessions` |
| Минимум ревьюеров: `medium` ≥1, `hard` ≥2; UI блокирует кнопку + объясняет | `ReviewerPickerModal` + `POST /api/review-sessions` |
| Сессия завершается автоматически когда все assignments `completed + approved` | `PUT /api/reviewer/assignments/[id]` |
| Общий чат: только ревьюеры текущей сессии + автор + админ; посторонние получают 403 | `GET/POST /api/sessions/[id]/review-comments` |
| Ревьюер не из текущей сессии не должен видеть чат другой сессии | access control в `/api/sessions/[id]/review-comments` |
