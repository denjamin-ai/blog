## Design Context

### Users

Разработчики разного уровня — от джунов до сеньоров. Читают в рабочее время и после: за ноутбуком, в тишине, с кофе. Приходят за глубиной, а не за хайпом. Ценят, когда автор уважает их время и не растекается мыслью.

Формат: длинные MDX-статьи с кодом, схемами, формулами. Интерфейс должен исчезать, оставляя только текст.

### Brand Personality

**Три слова:** Живой · Тёплый · С голосом

Блог ощущается как конкретный человек, который думает вслух и не боится иметь мнение. Не корпоративная безликость, не академическая сухость — скорее умный коллега, которому интересно то, о чём он пишет.

**Тон:** editorial/magazine — серьёзный, чистый, с характером. Ориентиры: Stripe Blog, Vercel Blog — но со своим лицом.

### Aesthetic Direction

**Визуальный тон:** refined editorial. Строгая сетка, достаточно воздуха, но не пустынность Medium. Типографика делает всю работу — орнамент минимальный и намеренный.

**Тема:** light + dark, обе равноценны. Аудитория использует оба режима; не делать dark «по умолчанию для крутости» и не делать light «для безопасности».

**Цвет:** синий акцент остаётся, но переходим на OKLCH. Нейтралы тонируются в сторону акцента (chroma 0.005–0.01). Акцент работает редко — ссылки, фокус, активные состояния. Не размазывать его по интерфейсу.

**Типографика:** нужны веб-шрифты с хорошей поддержкой кириллицы.
- Display/заголовки: шрифт с характером — editorial, с засечками или выразительный sans. **Не** из списка запрещённых (Playfair, Lora, Newsreader, Cormorant и др.). Кандидаты: Bitter (slab, editorial), Vollkorn (old-style serif, книжный), Spectral (элегантный, есть кириллица).
- Body: гуманистический serif или читабельный sans для длинного текста. Кандидаты: Literata (спроектирован для длинного чтения, кириллица), PT Serif (нейтрально, но надёжно), Wix Madefor Text.
- Mono: уже настроен (`ui-monospace / Cascadia Code / Source Code Pro`) — не трогать.

**Пространство:** 4pt-сетка, семантические токены (`--space-sm`, `--space-md` и т.д.). Ритм через контраст — заголовок с воздухом над ним читается важнее. Тело статьи: max-width ~68–72ch.

### Design Principles

1. **Типографика делает работу.** Иерархия строится размером, весом и пространством — не цветом, не декором. Если что-то выглядит плоско, решение — контраст в шрифте, не добавление элементов.

2. **Акцент работает потому, что он редкий.** Синий — только ссылки, интерактив, активные состояния. Разбавить = убить.

3. **Читатель пришёл за текстом.** UI исчезает. Никаких карточек ради карточек, никаких sidebar-ов в конкурирующей иерархии с основным текстом.

4. **Голос ощущается.** Дизайн читается как мнение, а не как шаблон. Шрифт с характером, неожиданный, но не случайный выбор — это и есть «своё лицо».

5. **Без корпоративного шаблона.** Не stock-иллюстрации, не одинаковые карточки icon + heading + text, не маркетинговый тон, не «наша команда экспертов».

### Anti-References

- **Medium** — пустое пространство как самоцель, большие паддинги ради паддингов, serif-статьи с аплодисментами
- **Dev.to / Hashnode** — гейминг-карточки, яркие теги, социальные счётчики на первом плане
- **Корпоративный tech-blog** — шаблонные иллюстрации, продуктовый тон, ощущение что писал маркетинг
- **AI-slop дизайн** — градиентный текст, glassmorphism, cyan-on-dark, «умные» карточки с иконками над каждым заголовком
